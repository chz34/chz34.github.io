## 简介

HPF（High Performance Framework）是一个高性能单机异构并行框架，目标是在单机上充分利用 CPU、GPU 以及各类 AI 加速卡等异构计算资源，实现高效的任务并行调度与执行。

## 核心设计目标

- **异构统一调度**：屏蔽不同计算设备的差异，提供统一的任务提交与调度接口
- **低调度开销**：最小化运行时调度延迟，保证算子级并行效率
- **灵活的依赖管理**：支持任务间数据依赖描述，自动推导并行执行图

## 架构概览

```
┌─────────────────────────────────┐
│           用户接口层             │
├─────────────────────────────────┤
│         任务图构建 & 编译        │
├──────────────┬──────────────────┤
│   CPU 调度器 │   设备调度器      │
├──────────────┼──────────────────┤
│  CPU Worker  │  GPU / 加速卡    │
└──────────────┴──────────────────┘
```

## 关键技术点

### 异构任务调度

框架将计算任务抽象为有向无环图（DAG），根据算子的设备亲和性自动将任务分配到合适的计算设备上执行。

### 内存管理

引入跨设备内存池机制，减少 Host/Device 间数据拷贝，降低内存分配开销。

### 同步原语

采用轻量级事件机制替代重量级锁，保证任务间同步的高效性。
